# Example: Landing Concept Workflow

## Task
Create a benchmark-aware landing page concept without defaulting to the first polished hero section.

## Divergence prompt
Generate 7 clearly different landing concepts.
Do not write final page copy yet.
For each concept provide:
- concept name
- hero worldview
- headline direction
- proof strategy
- CTA rhythm
- benchmark-fit note
- risk

## Selection prompt
Evaluate the concepts using:
- distinctiveness
- brand fit
- conversion logic
- scalability into sections
- derivative risk
Return top 2 only.

## Convergence prompt
Develop only #1 into:
- hero headline
- subhead
- 4 section arc
- proof placement
- CTA sequence
- benchmark safety note

## Anti-fixation check
Before finalizing, answer:
- Did we choose a real concept, not just a prettier hero?
- Would the structure still make sense if benchmark references were removed?
